# Androidアプリ開発テンプレート 2025冬版

## ユーザーとの接し方について（重要）

このプロジェクトは、Androidの技術的な知識が少ない方(ユーザー)でもアプリ開発ができるように作られたものです。

■ 応答時の注意点

- ドメインやUIなど挙動や見た目に関わることを中心にユーザーに質問するようにする
- 専門用語を使う場合は、簡単な説明を添える
- 手順は具体的かつ段階的に説明する
- 「なぜそうするのか」の理由も平易な言葉で伝える

■ 実装時の注意点
- ライブラリ選定やアーキテクチャ構成に関してはこのテンプレートの内容の則り、基本的にはエージェント側で決定するようにし、ユーザーに決定を委ねない
- ここに書いてないライブラリ等が必要な場合や既存のアーキテクチャーから逸脱することが想定される機能の場合は、ユーザーに平易な文章でメリット/デメリットを説明しWeb等で知識を補完してもらうことを推奨する
- 対して、ドメイン、ロジックの実装する際には、既存のコードベースが必ずしも正ではありません!!ユーザーの指示通り実装することを優先する

## ビルドコマンド

```bash
# ビルド（アプリをコンパイルする）
./gradlew assembleDebug

# クリーンビルド（古いファイルを消してから新しくビルド）
./gradlew clean assembleDebug

# リリースビルド（公開用のアプリを作成）
./gradlew assembleRelease
```

## プロジェクト構成

### 技術スタック
- **言語**: Kotlin 2.0
- **UI**: Jetpack Compose + Material Design 3
- **最小対応Android**: API 24 (Android 7.0)
- **ターゲットSDK**: 36

## 導入のライブラリ構成

### 依存性注入
- **Hilt**

### ネットワーク通信
- **Retrofit**
- **OkHttp**
- **Kotlinx Serialization**

### データ保存
- **Room**
- **DataStore**

### 画面遷移
- **Navigation Compose**

### 画像読み込み
- **Coil**

## アーキテクチャ

```
UI層（画面表示）
    ↓
Domain層（ビジネスロジック・アプリの核となる処理）
    ↓
Data層（データの取得・保存）
```

### ディレクトリ構成

```
app/src/main/java/com/example/apptemplate2025winter/
├── data/               # データの取得・保存を担当
│   ├── local/          # アプリ内保存（DB、設定）
│   ├── remote/         # サーバー通信
│   └── repository/     # データ取得の窓口
├── domain/             # アプリの核となる処理
│   ├── model/          # データの形を定義
│   └── usecase/        # 具体的な処理
├── ui/                 # 画面表示
│   ├── screens/        # 各画面
│   │   └── {画面名}/                  # 画面ごとにディレクトリを作成
│   │       ├── {画面名}Screen.kt      # 画面の見た目（Composable関数）
│   │       ├── {画面名}ViewModel.kt   # 画面のロジック・状態管理
│   │       └── {画面名}UiState.kt     # 画面の状態を表すデータクラス
│   ├── components/     # 再利用できる部品
│   ├── navigation/     # 画面遷移
│   └── theme/          # 色やフォントの設定
└── di/                 # 依存性注入の設定
```

※ 新しい画面を追加する際は、screens配下に画面名のディレクトリを作成し、
  上記3ファイル（Screen, ViewModel, UiState）をセットで作成すること
